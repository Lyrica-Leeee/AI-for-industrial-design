📤 如何实现真实上传功能

发布指南 · 让学生真正可以提交内容

静态 HTML 文件本身无法存储数据。要让学生真实提交案例并让其他用户看到，你需要将本网站配合一个在线表单后台使用。以下是两种最简单的方案：

推荐 · 最简单

方案 A：Airtable 表单

① 注册 airtable.com（免费）
② 创建 Base → 添加字段（姓名/标题/类型/工具/描述/附件）
③ 点击 Share → 创建公开表单链接
④ 将链接替换页面中所有 YOUR_AIRTABLE_FORM_URL_HERE
⑤ 老师在 Airtable 后台审核后，手动更新网页中的已展示案例

自动化 · 稍复杂

方案 B：Notion 表单

① 在 Notion 中创建数据库
② 安装 NotionForms 插件生成公开表单
③ 表单提交自动进入 Notion 数据库
④ 将表单链接替换页面中的占位符
⑤ 可设置审核工作流，审核通过后自动展示

1
部署网页到 GitHub Pages（免费）
注册 GitHub → 创建仓库 → 上传 id_ai_toolkit_v3.html → Settings → Pages → 选择 main 分支 → 保存。几分钟后网站会分配一个公开 URL，所有学生均可访问。
2
创建 Airtable 表单并替换链接
在 HTML 文件中搜索 YOUR_AIRTABLE_FORM_URL_HERE（共 14 处），替换为你创建的 Airtable 表单链接。推荐创建两个表单：一个用于案例/教程上传，一个用于讨论留言。
3
设置 Airtable 字段（建议）
案例表单字段：姓名/学号、标题、类型（单选）、所属阶段（单选）、涉及工具、描述、文件链接（学生可提交网盘链接）、状态（待审核/已发布）。Airtable 免费版支持最多 1,000 条记录，足够一学年使用。
4
可选：将审核通过的案例添加到网页
审核通过的学生案例，可手动添加到对应阶段的 .tut-grid 区域中（复制一个 tut-card 组件即可），然后重新上传 HTML 到 GitHub Pages 更新网站。
